# Istilah Penting dalam Git

Sebelum mulai menggunakan Git secara langsung, ada beberapa istilah penting yang perlu kamu pahami.

Istilah-istilah ini sering digunakan dalam Git, jadi penting untuk mengenalnya sejak awal agar tidak bingung saat praktik nanti.

---

## Repository

Repository (sering disingkat repo) adalah tempat penyimpanan project kamu di Git.

Di dalam repository, Git akan menyimpan:
- file project  
- riwayat perubahan  
- informasi lainnya terkait project  

Sederhananya, repository adalah “rumah” dari project kamu.

---

## Commit

Commit adalah proses menyimpan perubahan ke dalam repository.

Setiap commit biasanya disertai dengan pesan (commit message) yang menjelaskan perubahan apa yang dilakukan.

Contoh:
- menambahkan fitur login  
- memperbaiki bug  
- mengubah tampilan  

Commit membantu kamu melacak perubahan yang terjadi dari waktu ke waktu.

---

## Branch

Branch adalah cabang dari project utama.

Dengan branch, kamu bisa:
- membuat fitur baru tanpa mengganggu project utama  
- mencoba eksperimen  
- mengerjakan bagian tertentu secara terpisah  

Branch sangat penting dalam kerja tim.

---

## Clone

Clone adalah proses menyalin repository dari tempat lain ke komputer kamu.

Biasanya digunakan ketika:
- mengambil project dari GitHub  
- mulai bekerja dalam tim  

Setelah clone, kamu bisa langsung mengerjakan project tersebut di komputer kamu.

---

## Push

Push adalah proses mengirim perubahan dari komputer kamu ke repository (misalnya ke GitHub).

Artinya:
- perubahan yang kamu buat akan tersimpan di server  
- orang lain bisa melihat perubahan tersebut  

---

## Pull

Pull adalah proses mengambil perubahan terbaru dari repository ke komputer kamu.

Biasanya dilakukan ketika:
- ada perubahan dari teman satu tim  
- kamu ingin update project ke versi terbaru  

---

## Kesimpulan

Istilah-istilah ini adalah dasar yang akan sering kamu temui saat menggunakan Git.

Dengan memahami istilah ini, kamu akan lebih siap untuk mulai praktik menggunakan Git.
