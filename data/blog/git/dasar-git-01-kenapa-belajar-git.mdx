# Kenapa Harus Belajar Git?

Pernahkah kamu mengalami hal seperti ini?

Bayangkan kamu sedang mengerjakan sebuah tugas atau sebuah project.

Setiap kali ada perubahan, kamu tidak mengganti file yang lama, tetapi membuat file baru agar lebih aman.

Awalnya mungkin terasa tidak masalah.  
Namun lama-kelamaan, file kamu menjadi seperti ini:

- `tugas1.docx`
- `tugas_revisi.docx`
- `tugas_revisi_fix.docx`
- `tugas_revisi_fix_final.docx`

Semakin banyak perubahan, semakin sulit kamu menentukan:

- mana file yang paling terbaru  
- perubahan apa saja yang sudah dilakukan  
- dan versi mana yang seharusnya digunakan  

Alih-alih menjadi rapi, justru file kamu menjadi tidak teratur.

Bagaimana, pernah mengalami hal seperti itu?

---

Sekarang bayangkan jika kamu bekerja dalam tim.

Kamu dan teman-temanmu mengerjakan file yang sama.  
Masing-masing menyimpan versi sendiri, lalu saling mengirim file.

Akibatnya:

- file menjadi banyak versi  
- perubahan saling bertabrakan  
- dan sulit menentukan mana yang paling benar  

Tentu itu akan memakan banyak waktu, bukan?

---

## Di sinilah Git hadir sebagai solusi

Git adalah sebuah tools yang digunakan untuk mengelola perubahan file dalam sebuah project.

Dengan Git, kamu bisa:

- menyimpan riwayat perubahan file  
- melihat siapa yang melakukan perubahan  
- kembali ke versi sebelumnya jika terjadi kesalahan  
- dan bekerja sama dengan tim tanpa takut file tertukar  

Sederhananya, Git membantu kamu untuk:

> mengatur file project agar tetap rapi, terstruktur, dan aman.

---

## Kenapa Ini Penting untuk Dipelajari?

Dalam dunia IT, terutama programming, perubahan kode adalah hal yang pasti terjadi.

Tanpa sistem yang baik, kamu akan kesulitan:

- melacak perubahan  
- memperbaiki kesalahan  
- dan berkolaborasi dengan orang lain  

Oleh karena itu, Git menjadi salah satu skill dasar yang wajib dimiliki oleh seorang developer.
